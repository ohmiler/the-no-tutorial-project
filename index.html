Level 2: Comprehension

The "Why" Question

Dev ที่ดีไม่ใช่แค่เขียนโค้ดได้ แต่ต้องตอบได้ว่า "ทำไม" ถึงเขียนแบบนั้น

Architecture

ทำไมคุณถึงเลือกวางโครงสร้าง Folder แบบนี้? แยก Component vs Page ยังไง? มันดีกว่าแบบอื่นตรงไหน?

State Management

ถ้าต้องส่งข้อมูลข้าม Component คุณใช้วิธีไหน (Props, Context, Redux, Zustand) และทำไมถึงเลือกวิธีนั้นในโปรเจกต์นี้?

Dependencies

Library ตัวนี้ที่คุณ `npm install` มา คุณรู้ไหมว่ามันทำงานยังไงคร่าวๆ หรือแค่ลงเพราะเขาบอกให้ลง?

ห้ามตอบว่า "ไม่รู้ครับ เห็นเขาทำกันแบบนี้"
Level 3: Problem Solving

The Troubleshooter

เมื่อหน้าเว็บขาวโพลน (White Screen of Death) หรือ API Error 500 คุณทำยังไง?

1

F12 Console

ดู Error Log สีแดง อ่านข้อความ Error ให้เข้าใจ

2

Network Tab

เช็ค Request ว่าส่งไปไหม Response กลับมาเป็นอะไร

3

Backend Log

ถ้า Frontend ปกติ ให้ไปดูที่ Server หรือ Database

4

Google It

นำ Error Code ไปค้นหา และประยุกต์ใช้คำตอบ

The "No-Tutorial" Project

Mission: สร้างสมุดรายชื่อพนักงาน (Employee Directory)

กฏเหล็ก (Rules)

ห้ามดูคลิปสอนทำโปรเจกต์นี้ตั้งแต่ต้นจนจบ (Tutorial) แต่สามารถเปิด Docs หรือ Google หาคำตอบเฉพาะจุดได้

ฟีเจอร์ที่ต้องมี (Requirements)

  • Create & Read: หน้าแรกแสดงรายชื่อพนักงานทั้งหมด (Card หรือ Table) และมีปุ่ม "เพิ่มพนักงาน" ที่ใช้งานได้จริง
  • Update & Delete: กดแก้ไขข้อมูลพนักงานได้ และกดลบพนักงานออกจากระบบได้
  • Search & Filter: ช่องค้นหาชื่อต้องทำงาน (Real-time หรือกด Enter) และมี Dropdown กรองแผนก (IT, HR, Sales)
  • Tech Stack: ใช้ Framework ที่คุณถนัด (React, Vue, หรือ HTML/JS ธรรมดา) ข้อมูลเก็บใน LocalStorage หรือ Mock API
